## 🔄 개발 워크플로우

### 1. 브랜치 전략
```
main (프로덕션)
├── develop (개발)
├── feature/frontend
├── test/ui
```

### 2. 작업 순서
1. **각자 담당 브랜치에서 개발**
2. **자체 테스트 완료**
3. **develop 브랜치로 PR 생성**
4. **코드 리뷰 및 병합**
5. **통합 테스트 실행**

### 3. 커밋 메시지 규칙
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 스타일 변경
- refactor: 리팩토링
- test: 테스트 추가/수정

### 4. PR 규칙
- 제목: [Role] 간단한 설명
- 설명: 변경 사항 상세 기술
- 리뷰어: 최소 1명 이상
